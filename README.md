LeafNote は、思考をクラウドに預けず、自分の手元で育てるためのローカルファーストなノートです。

軽いメモ、設計の下書き、調査ログ、タスク、コード、表、画像、Mermaid 図。
散らばりがちな作業の断片を、1つの場所に静かに集めて、あとから読み返せる形に整えてくれます。

派手な同期機能やアカウント連携よりも、「すぐ書ける」「手元に残る」「Markdown として外へ出せる」ことを大切にしたノートです。

## 書くことに集中できる

LeafNote の魅力は、ツールを使うための準備がほとんどいらないことです。
重いワークスペースを作る必要も、毎回ログインする必要もありません。

開けば、そこはもう自分のノートです。
小さなメモから長い設計書まで、思いついた順に書き始めて、あとからブロックとして整えられます。

## Markdown の自由さを失わない

ブロックエディタは便利ですが、データが閉じ込められる感覚があると長く使いにくくなります。
LeafNote は Markdown を自然に扱えるので、書いた内容を別の場所へ持ち出しやすく、既存の Markdown メモとも付き合いやすい設計です。

自分の文章が、特定のサービスや形式に縛られすぎない。
それだけで、日々のメモはかなり気楽になります。

## 設計メモに強い

LeafNote は、ただの文章置き場ではありません。
見出し、リスト、ToDo、引用、コード、表、画像、コールアウトを混ぜながら、考えを構造化できます。

さらに Mermaid 図を扱えるので、フロー、状態、関係性、スケジュールのような「文章だけでは伝わりにくいもの」も同じノートの中に置けます。
エンジニアの仕様メモ、個人開発の構想、調査結果の整理と相性が良いノートです。

## 手元にある安心感

クラウドノートは便利ですが、すべてのメモを外に置きたいわけではありません。
まだ固まっていないアイデア、仕事前のラフな整理、公開するほどではない個人の知識。

LeafNote は、そういう途中の思考をローカルに置いておくための道具です。
誰かに見せる前の、まだ柔らかい考えを安心して書けます。

## ノートが育っていく

LeafNote は、単発のメモだけでなく、積み上がっていくノートに向いています。
階層で整理し、検索で呼び出し、必要なときに Markdown として取り出せる。

雑に書き始めても、あとから形にできる。
その余白があるので、日々のメモが少しずつ自分だけの知識ベースになっていきます。

## こんな人に刺さります

- Notion ほど大きくなくていいけれど、普通のメモ帳では物足りない人
- Markdown で書きたいけれど、見た目よく整理もしたい人
- 個人開発、設計、学習、調査のメモをローカルに残したい人
- 思考途中のメモをクラウドに置くことへ少し抵抗がある人
- 1つのファイルで持ち運べる、軽い作業場が好きな人
- 図、コード、表、タスクを同じ文脈の中で扱いたい人

## LeafNote が目指すもの

LeafNote は、巨大な情報管理システムではありません。
もっと小さく、もっと手元に近いノートです。

書きたいときにすぐ開ける。
考えを邪魔せず受け止める。
必要になったら外へ持ち出せる。

毎日の作業のそばに置いておける、静かで頼れる Markdown ノート。
それが LeafNote です。
